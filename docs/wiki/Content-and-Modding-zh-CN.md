# 内容与模组

> **Wiki 页面版本：** zh-CN 1.0.0 · **英文源版本：** EN 1.0.0 · **产品基线：** v0.1.3 · **更新日期：** 2026-07-18
> **English:** [Content and Modding](Content-and-Modding) · **同步状态：** 已同步至 EN 1.0.0

Alohayo World 坚持配置优先。JSON 可以选择已注册的引擎能力，但不能执行代码。带命名空间
的稳定 ID 是地形、角色、物品、地图、存档、本地化与未来插件之间的共同语言。

## 内容包

内容包声明 ID、版本、依赖、文件所有权和导出内容。依赖顺序是确定性的；缺失或循环依赖
会失败。自定义覆盖保留来源包、冲突诊断和解析哈希，以支持存档兼容。

## 添加地形

1. 在 `content/core/biomes.json` 添加稳定 ID 与数值定义。
2. 在 `terrain-rules.json` 添加现实说明、生成条件、表面层、物理和变换规则。
3. UI 使用前先添加英文和简体中文名称/说明。
4. 更新地形、GIS、地图文档和两种语言 Wiki。
5. 添加确定性生成和本地化覆盖测试。
6. 运行内容校验，并在开发地形展示图中检查。

大陆、岛屿、河流、海湾和悬崖不应因为“可见”就被编码成气候生物群系；应根据含义使用
拓扑或特征覆盖层。

## 添加自定义区域

把区域 JSON 放入 `content/maps/<pack>/areas/` 并在该包地图索引注册。优先使用紧凑的
patch、几何形状、实体/保护区声明与生成修饰器，不要枚举成千上万个格子。开发测试区域
默认关闭，不能污染普通种子。

## 添加角色或装备

先添加本地化键，再扩展能力、身份、原型、外观池、槽位、物品或规则目录。只引用稳定
地形 ID 和能力标签；玩家、NPC、敌人保持同一数据模型。校验需求、池、槽位兼容与确定性。
可选派生逻辑放在纯 `character-rules` 包。

## 兼容规则

- 优先添加稳定 ID；删除或改变含义需要存档迁移。
- 合并顺序与冲突必须显式且确定。
- 可见字符串只能来自 `i18n/`。
- 资产必须原创/生成或经核验 CC0，并登记来源、许可、修改和 checksum。
- 配置只能选择注册行为，不能包含 JavaScript 或 Rust 代码。
- 内容包不能根据翻译名称、颜色或贴图推断地形物理。

常用命令：`yarn validate:content`、`yarn validate:assets`、`yarn validate:wiki`、
`yarn test`、`yarn build`。
