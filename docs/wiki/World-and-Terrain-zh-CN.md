# 世界与地形

> **Wiki 页面版本：** zh-CN 1.1.0 · **英文源版本：** EN 1.1.0 · **产品基线：** v0.1.3 · **更新日期：** 2026-07-23
> **English:** [World and Terrain](World-and-Terrain) · **同步状态：** 已同步至 EN 1.1.0

地图是核心模拟模型。地形由连续地理字段推导，而不是先绘图再补理由。同一组稳定字段同时
服务于探索、道路、聚落、生物、天气、战斗位置与自定义区域。

## 地理分层

| 层        | 示例                                  | 权威模块         |
| --------- | ------------------------------------- | ---------------- |
| 连续字段  | 海拔、湿度、温度、坡度                | 确定性生成器     |
| 水文      | 填洼高程、D8 流向、汇流量、流域、洼地 | map/Wasm 栅格    |
| 拓扑      | 海洋/湖体、大陆/岛屿身份、别名        | 拓扑解析器       |
| 地貌过程  | 侵蚀潜力、沉积物、堆积、洪泛区        | 确定性元数据     |
| 地形/材质 | 森林、沙漠、湿地、高山、冰川          | 配置驱动分类     |
| 特征      | 河流、道路、聚落、自定义覆盖          | typed feature 层 |
| 表面      | 水、泥、雪、霜、灰、烧痕              | 可逆天气/动作层  |
| 表现      | 轮廓、过渡、细节、迷雾、光照          | 引擎渲染器       |

海洋/湖泊与大陆/岛屿属于拓扑；河流是跨格子的线性水文特征；海湾、海峡、三角洲、洞穴、
半岛、悬崖和山谷属于地貌/特征层。分层避免重复 biome，并让玩法能提出精确查询。

## 地形物理契约

每个地形都必须定义现实描述、世界内行为、生成权重与条件、允许表面层、移动/操控/暴露/
进入能力/危险，以及可破坏性和变换方法。生物栖息标签、聚落适宜度、道路成本和调色板
也由配置提供。机器权威是 `terrain-rules.json` 与 `biomes.json`，中英文缺一不可。

## 表面层与变换

天气和动作先添加临时表面：雨产生积水/泥，雪覆盖道路/树冠，融雪形成泥雪，热量融冰，
火留下烧痕。只有规则明确说明底层材质变化时才改变地形 ID。移除树木或积雪不等于删除
土壤；开矿不等于把裸岩变成草地；短期洪水也不立刻创建永久三角洲。

## 流式与开发测试

区块由生成器版本、种子、坐标、大小和内容解析结果唯一决定。加载顺序、负坐标和驱逐/
重载不能改变结果。`core:terrain-showcase` 可在起点附近放置全部 26 种地形，用于 i18n、
渲染、移动、道路和天气测试；它默认关闭，不属于正常世界。

## 水域地形与区块接缝

水域有三类不能混为一谈的权威：拓扑解析器决定水体是否连通海洋或为内陆湖泊；水文栅格
决定本地区域 D8 流向、汇流量与局部流域；渲染提示决定岸线如何柔化表现。

当前岸线字段是带符号的局部距离：水为负值、陆地为正值、`0` 接触水陆边界，`+/-127`
表示本区块内没有岸线。它只为轮廓渲染下方提供轻微水面材质，不会改变地形 ID、移动或
拓扑身份。

每个区块还发布 `ChunkDrainageSummary`，记录跨越四个方向边界的局部水流交接。其状态始终
是 `provisional`，尚未拥有全世界稳定流域身份。#38 负责 halo、接缝协调、河流图、持久化与
加载顺序基准；#41 负责 halo 感知岸线字段、GPU 迷雾与湖泊/河口/三角洲材质。局部字段不能被
当作“河流在区块边缘结束”的证据。
